Introduction

This is a living document that represents the guidelines for each individual UI component in the [app_name] app.

Purpose

The purpose of this style guide is to visualize all components that are 100% developed within [app_name]. See Components section for all available components.

Benefits

Displaying the components in the style guide will allow [company_name] to:

  1. Visualize style updates
  2. Provide code samples
  3. Apply themes across components

Usage

New Components

For every new component created within the [app_name] project, we want to:

  1. Create a new .html file in the following directory and with the markup output from the new component: 
    /src/materials/components/
    Example CSS: 
              <button class="button">Button Text</button>

  2. Create a new _*.scss file in the following directory and the relative styles in the _*.scss file:

    (Don't forget the _ or underscore in the file name since it's a partial) 
    /src/assets/toolkit/styles/
    Example SCSS: 
    
          .button { ... }
  3. Import the new _*.scss file (step 2) into the following file and with the following code: 
    /src/assets/toolkit/styles/toolkit.scss
    Example SCSS: 
    
          /*----------------------------------
           COMPONENT STYLESHEETS (CUSTOM)
          ------------------------------------*/

          @import 'button';

Updates & Maintenance

When a change is made to any component within the [app_name] project, the development team needs to make sure that the new markup and styles are also updated in this style guide app.

HTML changes

Update the *.html file with the newly updated html output from [app_name]

SCSS changes

Update the _*.scss file with the newly updated styles from [app_name]

Improvements

  • Use WebPack in [app_name] to built to the stylesheet output directory to elimiate need to update and maintain styles for this style guide. 
    /src/assets/toolkit/styles/
  • Use WebPack to build out themes for the style guide
  • Create a control that allows a user to change the theme of this style guide
